# FCM Create Helper SOP

## 适用场景

用于批量生成 FCM 草稿图。

适合场景：

- 每周按国家批量做 FCM
- 已经有固定 `FCM模版`
- 已经有歌曲信息表
- 人物图需要批量灌版，再人工微调

## 使用前准备

### 1. 模板准备

Figma 模板根节点建议命名为：

`FCM模版`

模板中请保留这些图层名：

- `人物`
- `歌名`
- `歌手名`
- `Hires标志`
- `背景`
- `大圆`
- `小圆`

### 2. CSV 准备

CSV 建议字段如下：

```csv
id,song,artist
001,Il cielo in una stanza,Gino Paoli
002,BLITZ,Niko Pandetta
003,Giovani Re,Sfera Ebbasta
```

说明：

- `id` 用于匹配图片序号
- `song` 对应歌名
- `artist` 对应歌手名

### 3. 图片准备

图片文件名需与 CSV 中的 `id` 对应，例如：

```text
001.jpg
002.webp
003.png
```

支持混合导入：

- 普通原图
- 你自己已经抠好的透明 PNG

如果勾选了“试验版本地去背”：

- 透明 PNG 会自动跳过去背
- 普通原图会尝试本地去背

## 标准操作流程

### 1. 打开 Figma 模板页

- 选中 `FCM模版`
- 或保证当前页面中存在一个叫 `FCM模版` 的模板节点

### 2. 打开插件

- 运行 `FCM Create Helper`

### 3. 导入 CSV

- 上传包含 `id / song / artist` 的 CSV

### 4. 导入人物图

- 上传与 `id` 对应的图片
- 可以混合上传普通图和透明 PNG

### 5. 选择是否本地去背

建议：

- 普通人物原图较多时，可以勾选 `试验版本地去背`
- 如果这批图里有些你已经手工抠好，就直接混着传，插件会自动识别透明图
- 如果某批人物边缘要求很高，可以不勾选，先用外部工具去背后再导入

### 6. 点击生成草稿

插件会自动完成：

- 复制模板
- 替换 `歌名`
- 替换 `歌手名`
- 替换 `人物`
- 自动生成 `背景 / 大圆 / 小圆` 初始颜色
- 按网格批量排版生成

## 生成后的人工检查

建议重点检查这几项：

- 人物位置是否合适
- 局部去背是否需要手修
- 调色是否符合人物衣服和整体感觉
- 极短歌名或超长歌名是否还要人工微调

## 推荐工作流

最推荐的实际工作流：

1. 用 CSV 整理好每个国家的歌名和歌手名
2. 下载人物图并按序号命名
3. 已经手工抠好的图直接保留透明 PNG
4. 其余图片交给插件先起一版
5. 批量生成后统一修人物位置、个别去背和调色

## 注意事项

- 本地去背属于试验功能，复杂眼镜、发丝、饰品仍可能需要人工修
- 插件更适合“批量起稿”，不适合代替最终审美判断
- 最终成图仍建议人工过一遍
